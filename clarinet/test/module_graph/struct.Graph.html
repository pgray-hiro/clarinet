pub struct Graph {
    handler: Arc<Mutex<dyn SpecifierHandler>>,
    maybe_tsbuildinfo: Option<String>,
    modules: HashMap<ModuleSpecifier, ModuleSlot>,
    redirects: HashMap<ModuleSpecifier, ModuleSpecifier>,
    roots: Vec<ModuleSpecifier>,
    roots_dynamic: bool,
    maybe_lockfile: Option<Arc<Mutex<Lockfile>>>,
}

A dependency graph of modules, were the modules that have been inserted via the builder will be loaded into the graph. Also provides an interface to be able to manipulate and handle the graph.

Fields

handler: Arc<Mutex<dyn SpecifierHandler>>

A reference to the specifier handler that will retrieve and cache modules for the graph.

maybe_tsbuildinfo: Option<String>

Optional TypeScript build info that will be passed to tsc if tsc is invoked.

modules: HashMap<ModuleSpecifier, ModuleSlot>

The modules that are part of the graph.

redirects: HashMap<ModuleSpecifier, ModuleSpecifier>

A map of redirects, where a module specifier is redirected to another module specifier by the handler. All modules references should be resolved internally via this, before attempting to access the module via the handler, to make sure the correct modules is being dealt with.

roots: Vec<ModuleSpecifier>

The module specifiers that have been uniquely added to the graph, which does not include any transient dependencies.

roots_dynamic: bool

If all of the root modules are dynamically imported, then this is true. This is used to ensure correct --reload behavior, where subsequent calls to a module graph where the emit is already valid do not cause the graph to re-emit.

maybe_lockfile: Option<Arc<Mutex<Lockfile>>>
